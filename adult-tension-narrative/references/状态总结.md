## 原则

存档必须让一个新实例直接重建当前状态，而不是重放历史。载入存档时不得重新执行完整开局、重新抽取角色或改变已提交结果。

“载入存档”和“解除暂停”是两个不同命令：

- `载入存档 / 续玩存档`：读取带有顶层 `save_version` 的 YAML。
- `解除暂停 / 恢复场景`：把已经载入的游戏从暂停状态恢复，不读取存档。
- 用户只说“恢复”且语义不清时，先询问是哪一种；不推进回合。
- 输入中出现顶层 `save_version` 时，始终优先识别为存档。

以下字段不可压缩：角色稳定 ID、明确年龄、世界常量、玩家硬边界、逐场景逐参与者的同意记录、安全状态、模式、Tier、追算开关、回合号、NPC 常量、自主冷却、pending 事件、远程钩子和最近一次完整 checkpoint。

台账只用于后台校准，不定期展示。只有玩家明确要求“存档”时才输出完整 YAML；“查看状态”只输出简短人话说明。

## 同意记录

同意记录必须绑定当前场景和具体参与者，不能只保存一个全局“已同意”。

1. 场景、地点或参与者发生变化时生成新的稳定 `scene_id`。
2. 每条许可使用稳定 `consent` ID，并保存其 `scene_id`、参与者 ID、明确范围、状态和最后确认回合。
3. `status` 只能是 `unknown|granted|withdrawn|not_applicable`。
4. 只有覆盖当前 `scene_id`、全部当前参与者和当前行为范围的 `granted` 记录才有效。
5. 新参与者、新行为、新场景、撤回、暂停或安全词都会使旧许可不能用于升级当前场景。
6. 不删除撤回记录；将其改为 `withdrawn` 并保留原 `scene_id`，以保留因果历史。

## 后台校准

轻量校准每回合检查年龄、边界、同意、安全状态、玩家控制权、时间一致性、事件去重和信息差，并将变化字段写入 `checkpoint.changed`。

完整校准在 `turn % 5 == 0` 或 `force_full=true` 时执行，重建当前状态、关系边、事件队列和 NPC 自主冷却，然后更新 `last_full_turn` 与 `next_full_turn`。场景变化、关系跃迁、事件兑现、快进、边界或同意变化、暂停/解除暂停、新 NPC、关键线索公开、自主行为、存档和载入都会提前触发完整校准。

校准结果不输出给玩家；它只影响下一次正文、存档和载入。

## 保存检查

1. `save_version` 必须等于 `3`，根节点必须是合法 YAML 映射。
2. 玩家和每个 NPC 都必须有唯一稳定 ID，年龄必须是明确且大于等于 18 的整数；布尔值不算整数。
3. 角色、边界、同意和事件 ID 在全局范围内不得重复。
4. active 边界必须有非空主题；边界状态只能是 `active|revoked`。
5. 每条同意记录必须绑定当前场景中的至少两名已知参与者，并有明确范围。
6. `safety_state` 只能是 `running|paused|stopped`。
7. 每个 NPC 必须保存完整自主冷却，关系信任值必须在 -5..+5。
8. 事件状态只能是 `pending|resolved|cancelled`；pending 事件必须有非空触发条件或到期时间。
9. 当前节点只能引用已存在的角色，并且必须能明确接续而不替玩家作关键决定。
10. `checkpoint.last_full_turn` 不得大于当前回合；四项 invariants 必须存在且为布尔值。

保存后，在 Skill 根目录运行：

```powershell
python scripts/validate_state.py path/to/save.yaml
```

缺少 YAML 解析库时先运行：

```powershell
python -m pip install PyYAML
```

## 载入流程

1. 检测顶层 `save_version`；含该字段的输入不得解释为“解除暂停”。
2. 若为 v2，先按下节迁移成 v3；未知版本停止载入并说明原因。
3. 运行完整保存检查；任何错误都先报告并修复，不生成正文。
4. 用存档重建世界、玩家、NPC、关系、许可和事件队列。
5. 不重新掷骰，不重新生成角色，不改变已提交结果。
6. 从 `current_node.unresolved_action` 前一刻继续。
7. 若 `safety_state` 不是 running，只允许非亲密叙事或元指令。
8. 载入本身不增加回合，不重述完整存档。

## v2 迁移到 v3

按以下顺序迁移，不得静默猜测：

1. 把 `save_version` 改为 `3`。
2. 若旧存档使用 `players` 列表，只在列表恰好有一项时改名为 `player`；否则停止并要求用户确认玩家主体。
3. 生成稳定 `consent.scene_id`。旧版全局 `consent.scene` 无法证明具体参与者和范围，因此不要自动迁移成 `granted`。
4. 将 `consent.grants` 初始化为空列表，把 `meta.safety_state` 设为 `paused`，并要求玩家重新确认当前场景参与者和许可范围。
5. 保留角色、边界、事件、关系、冷却、世界常量和 current_node，不重新生成 ID。
6. 缺少的冷却字段采用保守值：`last_turn=null`、`recent_turns=[]`、`cooldown_until=当前回合`。
7. 无法可靠推断的年龄保留为 `null`，停止相关内容并要求明确成年年龄；不得默认填 18。
8. 迁移后运行 v3 完整校验，通过后才允许接续。

v1 或未知结构没有足够稳定的字段映射，不自动迁移。保留原文，列出缺失字段并等待用户确认。
