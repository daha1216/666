# 世界运转

## 时间

保存当前时间、上次定位时间和本次 `delta_t`。玩家明确指定时间时直接采用；否则取完成动作所需的最小合理时间，不擅自快进。

小时间跨度只处理时钟、短期情绪和自然衰减。过夜、数日或明确快进时按 Tier 追算。

## Tier

| Tier | 追算范围 |
|---|---|
| 1 | 时间、持续状态、死线和定时事件 |
| 2 | Tier1 + 活跃 NPC 离屏行动 + 世界事件 |
| 3 | Tier2 + 关系网络传播、组织反应和远程后果 |

## 离屏行动

每个活跃 NPC 最多生成一行：

```text
<npc_id>｜<时间段>｜<地点>｜<行动>｜<留下或改变的事件ID>
```

行动必须来自该角色的身份、目标、已知信息和资源，不得重新抽取身份。离屏行动不能替玩家作出关键选择。

## 事件结构

```yaml
- id: evt-001
  source: turn-3
  created_turn: 3
  kind: timed
  trigger: "2026-07-13 08:00"
  due_at: "2026-07-13 08:00"
  status: pending
  consequence: "审议会要求公开答复"
  hook: false
```

`kind` 使用 `immediate|near|far|timed|probabilistic`。已解决事件改为 `resolved`，被明确取消则改为 `cancelled`，不得删除后重新创建同一含义事件。

## 处理顺序

1. 更新时间。
2. 兑现到期定时事件。
3. 兑现满足条件的 near/far 事件。
4. 处理持续状态自然变化。
5. Tier2 生成离屏行动和世界事件。
6. Tier3 沿关系边传播信息。
7. 以事件 ID 去重并提交。

## 关系图

```text
source_id -> target_id | 类型 | 渠道 | 信任(-5..+5) | last_updated_turn
```

传播速度由渠道、信任和时间跨度共同决定。未建立边的角色之间不得自动传播精确信息。

| 渠道 | 速度 | 失真 |
|---|---|---|
| 同住或同组 | 数小时 | 低 |
| 同事组织 | 当天 | 中 |
| 私人通信 | 视可达性 | 低至中 |
| 流言网络 | 1-3 天 | 高 |
| 专业链条 | 视流程 | 低 |
| 敌对监视 | 视投入 | 低，但可能故意误导 |

## 压缩

只压缩已解决事件和不再驱动人物反应的旧记忆。始终保留：

- pending 事件
- `[钩]` 远程事件
- 活跃死线
- 角色自主冷却
- active 玩家边界
- 当前场景同意和安全状态
- 关系图活跃边
